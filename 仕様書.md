## 友だち追加時の自動メッセージ（4通・即時）

1. 挨拶テキスト（`{Nickname}` / `{AccountName}` = 表示名 / TacTeQ）
2. お見積り用写真撮影ガイド（画像）
3. オレンジ Flex ボタン「お問合せフォーム入力」→ Harness LIFF フォーム（3ステップ）
4. 注意書き（URL なし）

### 入力ステップ（全6画面）

1. **今回のご相談**（傷の修繕 / 劣化 / リフォーム / 床鳴り / 塗装 / その他）
2. **対象箇所**（**複数選択可**。相談種別で出し分け）
   - 「自由入力」「その他」を選んだ場合、詳細テキストが **対象箇所** 欄にそのまま反映される（「自由入力」という固定文字は保存しない）
3. **お客様・物件**（業者/個人、住居種別、**原因は？**、工事中かどうか）
   - 住居種別: 持家（戸建て） / 持家（マンション） / 賃貸・借家 / 店舗
   - 原因は？: 業者様がお客様先で傷をつけた / お客様ご自身で傷をつけた / 経年劣化 / その他
4. **基本情報**（お名前、フリガナ、郵便番号→住所自動入力、市区町村）
5. **ご希望・スケジュール**（一番の目的、完了期日、作業希望日、ご要望、初めてか等）
6. **連絡・確認**（連絡手段は複数選択可、火災保険、個人情報同意チェック）
   - 連絡手段: LINE / 電話 / ショートメール / メール（「どれでも」は廃止）
   - 電話番号欄は常時表示。「お急ぎの方は電話番号を入力してください」の案内あり
   - 電話・ショートメール選択時 → 電話番号必須 / メール選択時 → メールアドレス必須

送信後、**トークルームに次の3通が自動送信**される（管理画面の個別チャットにも表示）:

1. **お問い合わせ内容**（Flexカード — 入力内容が一覧で表示）
2. 写真・**動画**送付のお願い（テキスト — 動画歓迎の文言あり）
3. お見積り用写真の撮影見本（ウェルカムと同じ画像 `welcome-estimate-photo-guide.png`）

### 送信通知（管理者 LINE）

フォーム送信時、登録済みの管理者 LINE ユーザーへプッシュ通知。

```bash
# 通知先 = TacTeQ 公式アカウントの友だちである LINE ユーザー ID（U で始まる）
TACTEQ_NOTIFY_LINE_USER_IDS=Uxxxxxxxx,Uyyyyyyyy pnpm tacteq:activate-form-notification
```

※ 管理者は事前に TacTeQ 公式 LINE を友だち追加しておくこと。

### Notion バックアップ

フォーム送信のたびに Notion データベースへ1行追加される。

| 項目 | 値 |
|------|-----|
| DB 名 | **LINE Harness 相談データベース** |
| Notion URL | https://app.notion.com/p/14b79bcdc6ae457fa8807128439d42ac |
| Database ID | `14b79bcd-c6ae-457f-a880-7128439d42ac`（D1 登録済み） |

旧 Notion フォーム DB（`TacTeQへお問合せ`）とは別。Harness LIFF フォーム専用。

**残り1ステップ（Worker から自動書き込み用）:**

1. [Notion Integrations](https://www.notion.so/my-integrations) で Internal Integration を作成（例: `TacTeQ LINE Harness`）
2. 「LINE Harness 相談データベース」をインテグレーションに **コネクト**
3. トークンを Worker に登録:

```bash
cd apps/worker && npx wrangler secret put NOTION_API_TOKEN
```

4. 接続テスト:

```bash
NOTION_API_TOKEN=secret_xxx NOTION_DATABASE_ID=14b79bcd-c6ae-457f-a880-7128439d42ac pnpm tacteq:activate-notion-backup --test
```

Notion 未設定時もフォーム送信自体は正常に完了する（バックアップのみスキップ）。

過去分の Notion 反映:

```bash
NOTION_API_TOKEN=secret_xxx pnpm tacteq:backfill-notion
```

**技術メモ:** 管理者通知・Notion バックアップは `executionCtx.waitUntil` で非同期完了させる（未設定だと Worker がレスポンス後に処理を打ち切る）。

### セキュリティ・整合性（2026-06）

| 項目 | 対応 |
|------|------|
| チャット status/メモ | `consolidateChatsForFriend` で重複 `chats` 行を統合。`in_progress` > `unread` > `resolved` で status をマージ |
| フォーム送信 | TacTeQ 問い合わせは **常に LIFF `id_token` 必須**（匿名 POST 不可）。副作用ありフォームも同様 |
| モバイル認証 | APIキーは保存せず **短命セッショントークン**（`lh_sess_*`、7日・ログアウトで失効）を `sessionStorage` に保持 |
| チャット CSV | `line_oa_chat_csv_*` は **`.gitignore` 対象**（PII をリポジトリに含めない） |
| CSV 日時 | インポート時 JST パースを `jstNow()` と同じ方式に統一（9時間ずれ修正） |

**注意:** 過去に Git に CSV をコミットした場合は `git filter-repo` 等で履歴から除去すること。ローカルフォルダはリポジトリ外で保管。

### 個別チャットからの送信

| 種類 | PC / iPhone | 備考 |
|------|-------------|------|
| テキスト | ○ | |
| 画像 (JPEG/PNG) | ○ | 最大 1MB（LINE preview 制限） |
| PDF | ○ | 最大 20MB。**リンク有効期限 30日**（`CHAT_PDF_TTL_DAYS` で変更可）。期限後は R2 から自動削除し URL は 410。LINE には **「PDFを開く」Flex ボタン**（期限の注記付き） |

**ストレージ制限（現状）**

| 項目 | 値 |
|------|-----|
| 1ファイル上限 | 20MB |
| リンク有効期限 | **30日**（アップロードから。Worker 環境変数 `CHAT_PDF_TTL_DAYS`） |
| 期限後 | URL は開けない（410）。6時間ごとの cron で R2 から削除 |
| アカウント全体の容量上限 | **コード上は未設定**（Cloudflare R2 のプラン枠に依存） |

## データ移行オペレーション

### 既存友だちの同期

```bash
pnpm tacteq:sync-followers          # D1 の line_accounts からトークン取得
pnpm tacteq:sync-followers --dry-run
```

LINE API `followers/ids` でフォロー中ユーザーを `friends` テーブルに upsert。

### チャット履歴インポート（LINE 公式アカウント管理画面 CSV）

LINE 管理画面からダウンロードした **1友だち1CSV** 形式（先頭3行メタ＋`送信者タイプ,送信者名,送信日,送信時刻,内容`）に対応。

1. チャット履歴をエクスポートしてフォルダに展開（例: `line_oa_chat_csv_260612_193545/`）
2. 友だち同期を先に実行（`pnpm tacteq:sync-followers`）
3. インポート:

```bash
pnpm tacteq:import-chat --dir /path/to/csv-folder
pnpm tacteq:import-chat --dir /path/to/csv-folder --replace   # 既存 csv_import 行を削除して再投入
pnpm tacteq:import-chat --file sample.csv --inspect   # 列確認
pnpm tacteq:import-chat --dir ... --dry-run
```

**2026-06-12 実施:** `line_oa_chat_csv_260612_193545` から 181 ファイル・約 5,011 件を `messages_log` に投入。

**2026-06-12 再インポート（JST 9時間ずれ修正）:** `--replace` で csv_import 由来データを削除後、同フォルダから再投入済み。

**制限:** テキストのみ復元（「写真を送信しました。」等のプレースホルダー）。画像・スタンプ・Flex は元ファイルなしのため不可。Harness 導入後の新規メッセージと重複する場合あり。
